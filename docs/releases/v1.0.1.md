# ScenarioLens v1.0.1 Release Notes

ScenarioLens v1.0.1 is a focused presentation-quality patch for the static
Scenario Explorer. It changes no autonomy metrics, baseline behavior, data
parsing, ranking, or real-data evidence.

## Fixed

- The open Reports menu now remains fully visible on narrow mobile screens.
- At widths below 900 px, the menu is anchored to the full navigation row
  instead of the Reports summary's horizontal position.
- The browser suite now opens the menu at 390 px, checks every report link,
  verifies its bounding box is inside the viewport, and repeats the geometry
  and page-overflow assertions at the 320 px supported minimum.

The patch clean-package gate passed 15/15 checks. Two independent builds
produced the same wheel SHA-256 digest:
`7d6c99b7605176e7f5f8ae524addf5a5f30f3a9649bd46d9c81362a676b4afff`.

## Why A Patch Release

The existing document-level overflow smoke remained green because absolutely
positioned content can sit outside the viewport without increasing document
scroll width. Manual release QA opened the menu and exposed the missing
interaction-level assertion. This patch fixes the CSS and preserves that exact
regression path in Playwright.

## Install And Run

Download the wheel attached to this release, then run:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install scenariolens-1.0.1-py3-none-any.whl
scenariolens demo --open
```

## Evidence Boundary

The 1,193-scenario full-corpus runs and 993-scenario frozen holdout remain the
same validated `1.0.0rc1` analytical evidence promoted in `v1.0.0`. This patch
does not rerun or relabel those results. Raw Waymo records remain outside git.
